<objective>
Implement the retry / backoff / quota classification policy for Phase 3:

1. `classifyError(err) → { kind, reason }` — pure classifier: `'retriable'` for 5xx + network errors, `'quota'` for 403 quotaExceeded, `'fatal'` for everything else (UploadError shape failures, OAuthError refresh failures, other 4xx, malformed responses).
2. `QuotaExceededError` — distinct error class so Plan 03-05's CLI handler can branch cleanly on `instanceof QuotaExceededError` for the "rerun tomorrow" remediation. Carries a `resumeAtHint: string` (ISO 8601 UTC) so the operator gets a concrete reset time.
3. `withRetry(fn, opts)` — generic wrapper: invokes `fn()`; on classify-retriable failure, sleeps for the next delay in `[1000, 4000, 16000]` ms and retries; on classify-quota failure, throws `QuotaExceededError` immediately; on classify-fatal failure, re-throws unwrapped immediately. After 3 retries (4 attempts total), throws the last error wrapped with `{ cause }`.
4. `publishWithRetry(args)` — convenience helper that calls `withRetry(() => uploadEpisode(args))`. This is what Plan 03-05 calls.

Purpose: Closes PUB-05 retry half (3 retries, 1s/4s/16s backoff, network-drop handling) and PUB-06 (quotaExceeded → fail loudly with "rerun tomorrow" hint; no publish.json written). The resumable-upload-resume-on-network-drop case is covered by googleapis SDK behavior — when the SDK's resumable upload encounters a network drop mid-PUT, it automatically resumes from the last committed chunk on the next attempt, which is exactly what `withRetry` provides by re-invoking `uploadEpisode`. (Resumability across retries works because `createReadStream(episodePath)` is invoked freshly on each retry — Plan 03-03's call site creates the stream inside the function body, not at args binding time. This is a subtle but load-bearing detail; this plan's tests pin it.)

Output: A retry policy fully tested against deterministic mocks (injectable sleep, injectable error sequences). `publishWithRetry` is the production entry point for Plan 03-05.
</objective>
