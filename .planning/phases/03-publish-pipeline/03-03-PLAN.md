<objective>
Implement the single-attempt YouTube uploader for Phase 3:

1. `uploadEpisode(args)` — the orchestrator that performs ONE `videos.insert` + ONE `thumbnails.set` against the YouTube Data API v3 via `googleapis` (Plan 03-01's dependency). Returns the `PublishRecord` Plan 03-05 will write to `.golazo/publish.json`.
2. Extend `src/publish/errors.ts` with `UploadError` for shape failures (missing videoId in response, missing required arg).
3. Export `PRIVACY_STATUS = 'unlisted' as const` — the typed defense-in-depth constant that pins the unlisted-only privacy literal at the module boundary. Used by the `videos.insert` requestBody AND consumed by Plan 03-05's `publishRecord.ts` for a compile-time cross-module binding.
4. Tests against `nock`-stubbed HTTP endpoints — verify request shape (title/description/privacyStatus), resumable protocol usage, thumbnail attached after videoId returned, and error paths.

Purpose: Closes PUB-01 (the upload + thumbnail attach + privacyStatus=unlisted + title/description applied). DELIBERATELY EXCLUDES retry, exponential backoff, quotaExceeded handling, and resumable-retry-on-network-drop classification — those concerns live in Plan 03-04's `withRetry` wrapper that wraps `uploadEpisode`. This layer is the "one HTTP attempt, properly shaped, raises typed errors" core; Plan 03-04 makes it production-resilient.

Output: A pure async function `uploadEpisode(args) → PublishRecord` exercised entirely against nock stubs in tests. No live network calls. Phase 3 manual smoke test (against a real throwaway channel) is a Phase 4 / operator action — out of scope for plan completion.
</objective>
