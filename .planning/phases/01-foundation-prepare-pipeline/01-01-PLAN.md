<!-- No existing codebase — this plan creates everything. Subsequent plans depend on the contracts below. -->

The commander.js subcommand registration pattern downstream plans will follow:

  Each `src/cli/commands/<name>.ts` exports a single function:
    `register<Name>Command(program: Command): void`

  The function calls `program.command(<name>)`, sets `.description(...)`, declares positional args via `.argument(...)`, then `.action(async (...) => { ... })`.

  The action body for unimplemented subcommands (render, publish, auth, all) prints the exact string `"<name>: not yet implemented"` to stderr and exits with code 2.

  The action body for `prepare` calls into the future `src/prepare/index.ts` orchestrator (which does not exist yet); for plan 01 it logs `"prepare: handler stub for <folder>"` and exits 0.

Node module convention: ESM only (`"type": "module"` in package.json), `.js` extensions on relative imports inside `src/` source files, NodeNext module resolution in tsconfig.
</interfaces>
