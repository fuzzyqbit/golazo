<!-- This plan creates the channels config contract. Downstream plans/phases will consume it. -->

ChannelConfig shape (matches the design spec example):

  type KidKey = string;  // resolved at parse time; common values 'leo', 'mateo'

  interface ChannelConfig {
    kid: KidKey;          // populated from the top-level yaml key
    name: string;
    club: string;
    jersey: number;       // integer 1..99
    accent: string;       // hex, validated against /^#[0-9a-fA-F]{6}$/
    source: string;       // free-form, e.g. "Veo", "Trace"
    youtube: {
      channelId: string;          // camelCased after parse; yaml key is `channel_id`
      oauthTokenPath: string;     // tilde-expanded absolute path; yaml key is `oauth_token`
    };
  }

  type ChannelsFile = Record<KidKey, ChannelConfig>;

API surface:

  loadChannelsFile(opts?: { path?: string }): ChannelsFile
    - default path: `./channels.yaml` (resolved against process.cwd())
    - throws ChannelsConfigError on any zod failure
    - throws ChannelsConfigError if the file is missing (path + 'create channels.yaml from channels.yaml.example' remediation)

  loadChannel(kidKey: KidKey, opts?: { path?: string }): ChannelConfig
    - calls loadChannelsFile, then validates kidKey is present
    - throws UnknownKidError when not present, listing valid keys
    - validates the OAuth token file exists at oauthTokenPath; throws ChannelsConfigError when missing

Error shape: every thrown error message MUST contain three substrings the test can grep:
  1. the field path (e.g. `leo.accent`, `mateo.youtube.oauth_token`)
  2. the reason (e.g. `must match #RRGGBB hex`, `must be an integer between 1 and 99`)
  3. a remediation hint (e.g. `edit channels.yaml and set leo.accent to a hex like #ffce5a`)

    Export const `CHANNELS_FILE_DEFAULT = './channels.yaml'`.

    Export `loadChannelsFile(opts: { path?: string } = {}): ChannelsFile`:
      1. Resolve path: `const filePath = resolve(opts.path ?? CHANNELS_FILE_DEFAULT)`
      2. If `!existsSync(filePath)`: throw `new ChannelsConfigError({ field: '(file)', reason: \`channels.yaml not found at ${filePath}\`, remediation: 'copy channels.yaml.example to channels.yaml and edit per-kid values' })`
      3. Read + yaml-parse; wrap parse errors in ChannelsConfigError with field `(yaml)`, reason `<parse error message>`, remediation `fix yaml syntax`
      4. Run `channelsFileSchema.safeParse(...)`. On failure, iterate `result.error.issues`, take the first issue, build a field path string like `leo.accent` or `mateo.youtube.oauth_token` from `issue.path.join('.')`, derive a reason from the issue code (`invalid_string` regex → `must match #RRGGBB hex`; `too_small`/`too_big` on jersey → `must be an integer between 1 and 99`; `invalid_type` on jersey → `must be a number`; missing key → `is required`), build remediation specific to the failing field. Throw ChannelsConfigError.
      5. On success: for each kid entry, build a `ChannelConfig` by camel-casing the yaml shape (`youtube.channel_id` → `youtube.channelId`, `youtube.oauth_token` → `youtube.oauthTokenPath`), tilde-expanding `oauth_token` (if it starts with `~/` or equals `~`, replace `~` with `homedir()`), and checking `existsSync(expanded)`. If the token file is missing, throw ChannelsConfigError with field `<kid>.youtube.oauth_token`, reason `oauth token file does not exist at <resolved path>`, remediation `run 'golazo auth <kid>' to create it`. Fill `kid: kidKey` on each entry. Return the assembled `ChannelsFile`.

    Export `loadChannel(kidKey: KidKey, opts: { path?: string } = {}): ChannelConfig`:
      1. Call `loadChannelsFile(opts)`
      2. If `kidKey` not in the resulting record, throw `new UnknownKidError({ kidKey, validKeys: Object.keys(file) })`
      3. Return the entry
