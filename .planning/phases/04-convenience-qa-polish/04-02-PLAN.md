    Edit vitest.config.ts. Keep the existing `test.include`, `test.globals`, `test.environment` exactly as-is. Replace the `coverage` block with:

    - `provider: 'v8'` (unchanged)
    - `reporter: ['text', 'html']` (unchanged)
    - `include: ['src/**/*.ts']` ‚ÄĒ the explicit denominator for the threshold; `src/` only, not remotion/ or tests/
    - `exclude: ['**/*.test.ts', '**/*.test-cases.ts', 'tests/fixtures/**', 'tests/snapshots/**', 'remotion/**', 'dist/**', '*.config.ts', '*.config.js', 'eslint.config.js', 'src/**/types.ts']`
    - `thresholds: { lines: 80 }`

    Rationale for excluding `src/**/types.ts`: type-only modules (e.g. `src/prepare/types.ts`, `src/config/types.ts`) export interfaces only and have no executable production logic; including them artificially deflates the denominator. If the executor finds a `types.ts` file that DOES contain runtime logic (e.g. a zod schema export), remove that specific exclusion or split the runtime portion out.

    Do NOT add `coverage.all: true` ‚ÄĒ that would force coverage on un-imported files, which is incompatible with the design spec's stance that Remotion components are excluded from line coverage. Leave `all` at its default (false): coverage is measured only on files that were imported during a test run.

    Use Context7 to confirm vitest 4.x coverage option names if any uncertainty: `npx --yes ctx7@latest library vitest "coverage thresholds include exclude"` then `npx --yes ctx7@latest docs <id> "coverage.thresholds.lines configuration vitest 4"`.
  </action>
