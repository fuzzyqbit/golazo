web/src/fonts.ts (NEW):

  import localFont from 'next/font/local';

  /**
   * Display font — Cormorant Garamond Italic.
   *
   * Source: remotion/assets/fonts/CormorantGaramond-Italic.ttf (the SAME
   * .ttf file Remotion compositions use; committed by Plan 02-01). next/font/local
   * resolves the `src` path relative to THIS file (web/src/fonts.ts) — so
   * `../../remotion/assets/fonts/CormorantGaramond-Italic.ttf` traverses up to
   * the repo root and into the shared assets dir. NO duplication.
   *
   * The Cormorant Garamond Italic weight-400 face is the same one Plan 02-01
   * documented in remotion/assets/fonts/README.md.
   */
  export const displayFont = localFont({
    src: '../../remotion/assets/fonts/CormorantGaramond-Italic.ttf',
    weight: '400',
    style: 'italic',
    variable: '--font-display',
    display: 'block',  // avoid FOUT on first paint — render nothing until font loads
    preload: true,
  });

  /**
   * Label font — Inter Regular + Bold.
   *
   * next/font/local accepts an array for multiple weight/style combinations.
   * Both Inter-Regular.ttf and Inter-Bold.ttf are committed under
   * remotion/assets/fonts/ by Plan 02-01.
   */
  export const labelFont = localFont({
    src: [
      {
        path: '../../remotion/assets/fonts/Inter-Regular.ttf',
        weight: '400',
        style: 'normal',
      },
      {
        path: '../../remotion/assets/fonts/Inter-Bold.ttf',
        weight: '700',
        style: 'normal',
      },
    ],
    variable: '--font-label',
    display: 'swap',  // body text — swap is fine; flash of fallback is acceptable
    preload: true,
  });

  /**
   * Convenience: concatenated CSS class names for both font variables.
   * Apply to <html> in the root layout so all descendant elements can use
   * `font-family: var(--font-display)` or `var(--font-label)`.
   */
  export const fontVariables = `${displayFont.variable} ${labelFont.variable}`;

  Notes on choices:
  - Relative path `../../remotion/assets/fonts/...` is from web/src/fonts.ts. Up two dirs lands at repo root; then descend into remotion/assets/fonts/. Context7 docs confirm next/font/local accepts relative paths (and recommend it for self-hosted fonts).
  - CSS variables (`--font-display`, `--font-label`) are the documented next/font idiom for sharing the font across many components via CSS.
  - `display: 'block'` for the display font, `'swap'` for the label font: display fonts (used in headlines) are visually disruptive when swapped, so we wait for them to load; body text uses fallback first because legibility matters more than typography fidelity for the brief preload window.
  - `preload: true` (both) — Next.js injects <link rel="preload"> for ATF (above-the-fold) fonts.

  Note on Next.js's font-source resolution AND "self-hosted" requirement:
  next/font/local copies / processes the TTF at build time. The relative
  path must work for BOTH dev (Turbopack) AND build (Turbopack production
  build). Turbopack's module resolution honours relative paths via
  bundler-style resolution; a relative path up two dirs out of the web
  subdir into a sibling `remotion/` dir is supported. If Turbopack rejects
  the path (e.g. because of an unstated bundler boundary at the workspace
  root), the FALLBACK is to copy the TTFs into `web/public/fonts/` and
  point next/font/local at `../../public/fonts/<file>`. Record any fallback
  in SUMMARY.

  Both the primary path (Turbopack copies the resolved cross-workspace TTF
  into `.next/static/media/`) and the fallback path (TTFs sit under
  `web/public/fonts/` and Turbopack still copies them into
  `.next/static/media/` or serves directly from /public/) satisfy UI-05's
  "self-hosted" intent — the requirement's "under `web/public/fonts/`"
  wording is descriptive of the fallback shape, not a hard constraint
  on Turbopack's output directory. SHA-256 byte-equality with the
  Remotion-side TTFs holds across both paths because both reference the
  same source bytes.

Locked decisions surfaced by this plan (record in SUMMARY):

  D-12: Font sharing mechanism = `next/font/local` with relative `src` path
        traversing up to `remotion/assets/fonts/` (primary), or copies
        under `web/public/fonts/` (fallback).
        Rationale: SAME .ttf bytes Remotion uses; NO duplication on the
        primary path; bundled by Turbopack at build time; explicit +
        verifiable. Alternatives considered and rejected:
        - Symlink web/public/fonts/ -> remotion/assets/fonts/: breaks
          Next.js's bundler resolution on some configurations; symlinks
          across git checkouts on Windows / WSL hit edge cases.
        - Copy the TTFs into web/public/fonts/ as the PRIMARY (not
          fallback) approach: byte-identical drift risk; two locations to
          keep synchronised. Only used as the fallback if Turbopack
          rejects the cross-workspace relative path.
        - Bare relative-import (e.g. `import font from
          '../../remotion/assets/fonts/...?url'`): doesn't use next/font's
          variable mechanism; loses preload + display optimisation.
        If next/font/local's path resolution fails at execution time,
        fall back to the copy approach (web/public/fonts/) and record
        the deviation. The byte-equality across paths is preserved because
        both reference the same source TTFs; UI-05's "self-hosted under
        `web/public/fonts/`" wording is interpreted as descriptive of the
        fallback shape (and of the broader "fonts served by this process"
        intent) — both primary and fallback paths satisfy that intent
        because both result in the web process serving the bytes itself.

  D-13: Web theme tokens are MIRRORED (manually duplicated), NOT imported
        from remotion/theme/tokens.ts.
        Rationale: the two trees compile under different tsconfig contexts
        (Remotion: React + NodeNext; web: Next.js + bundler). Cross-tree
        imports add tsconfig path mapping complexity for marginal benefit
        when the shared values are six color hex codes and three family
        strings. Drift detection lives in web/src/theme/tokens.test.ts
        cases #1-5 which pin the values to literals. If a future rebrand
        diverges the two trees deliberately, change the test; if
        accidentally, the test fails loudly.

  D-14: CSS Modules (not styled-jsx, not Tailwind, not vanilla-extract).
        Rationale: Next.js's default; zero deps; co-located with components;
        Turbopack supports them natively. Phase 7 (browse surface) may
        revisit if a more sophisticated solution becomes necessary; for
        Plan 04's six lines of CSS, Modules are right-sized.

  D-15: `display: 'block'` for display font; `'swap'` for label font.
        Rationale: matches the visual hierarchy — headlines should wait
        for the typographically-distinctive Cormorant Garamond Italic;
        body text can flash from fallback (system-ui) to Inter without
        visual jarring.
