<objective>
Convert the repo-root project into an npm workspace host so that `web/` (added in Plan 02) can import shared types from the root CLI package by name. This plan is the foundation prep: rename the root package from `golazo` to `@golazo/cli`, declare `workspaces: ["web"]`, verify all v1.0 CLI contracts (bin path, scripts, build output, 387 passing tests) survive the rename byte-for-byte.

Purpose: npm workspaces is the locked tech stack decision (per phase-planner context). The scoped name `@golazo/cli` is the import path Plan 02's `web/src/` will use (e.g. `import { manifestSchema } from '@golazo/cli/prepare/manifest.js'`). This plan must commit BEFORE any `web/` package exists — otherwise running `npm install` after Plan 02 would race against the rename. By isolating the rename into Wave 1, we surface any v1.0-breakage immediately and Plan 02 can build on a known-good workspace host.

Output: Root `package.json` with `name: "@golazo/cli"`, `workspaces: ["web"]`, all other v1.0 fields preserved exactly. A vitest unit suite at `tests/workspace.test.ts` pins the contracts the rename must NOT break (bin path, script names, type:module, packageManager). The v1.0 CLI is untouched on disk and 387 tests still pass.
</objective>
