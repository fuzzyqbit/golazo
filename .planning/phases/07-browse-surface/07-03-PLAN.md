<asset_route_contract>
<!-- Pinned thumb-serving stub. Phase 8 extends, does not rewrite. -->

Route: `web/src/app/api/asset/[kid]/[game]/thumb.png/route.ts`
HTTP: `GET` only (other methods return 405)
Runtime: `nodejs` (file system access)

Path safety rules — ALL of these MUST be enforced; integration test pins each:
1. Resolve `[kid]` and `[game]` from `params` (await per Next.js 16 contract)
2. If `kid` or `game` contains `/`, `\\`, `..`, or `\0` → return 403
3. Build candidate `path.join(resolveGolazoRoot(), kid, game, '.golazo', 'thumb.png')`
4. Call `path.resolve` on the candidate and on `resolveGolazoRoot()` and assert the resolved candidate startsWith `resolved-root + path.sep` (defends against symlink escapes)
5. If file does not exist → return 404
6. Return the bytes with `Content-Type: image/png` and `Cache-Control: private, max-age=60`

Phase 8 will add `episode.mp4/route.ts` next to this one, sharing the same `[kid]/[game]` dynamic segments and the same path-safety helper. The helper SHOULD be extracted to `web/src/lib/ui/assetPath.ts` (or similar) so both routes share it — name it carefully so Phase 8 is a drop-in extension.

THE STUB IS NOT A SHORTCUT — it is a partial implementation of PLAY-04 (path safety) that Phase 8 extends for PLAY-03 (full asset serving) + PLAY-05 (Range). Doing the path-safety correctly NOW saves Phase 8 from refactoring.
</asset_route_contract>

<server_client_split>
<!-- Pinned architecture per the planning_context decision. -->

`web/src/app/page.tsx` — Server Component (default async export):
1. `const sp = await searchParams; const params = parseListParams(sp);`
2. `const runtime = await getDiscoveryRuntime();`
3. `const accents = await getChannelAccents();`
4. `const allRows = queryAllEpisodes(runtime.cache);`
5. `const rows = applyListParams(params, allRows);`
6. If `allRows.length === 0` → render `<EmptyState rootPath={runtime.rootPath} />`
7. Else render `<EpisodeList rows={rows} accents={accents} params={params} totalCount={allRows.length} />`

`web/src/components/EpisodeList.tsx` — `'use client'` directive at top; receives rows + accents + params + totalCount as props (all JSON-serializable EpisodeIndex fields are primitives + nulls → safe to pass from Server to Client without dehydration concerns); uses `useRouter` from `next/navigation`; on sort dropdown change OR chip click, calls `router.replace(\`?\${serializeListParams(newParams)}\`, { scroll: false })`. The Server Component re-runs on URL change (Next.js dev re-renders the Server tree), so the client component does NOT itself re-sort — it just drives the URL and lets the server re-render the rows. This avoids duplicating sort logic on the client and avoids hydration mismatches.

CRITICAL: do NOT call `getDiscoveryRuntime()` from the client component. The Server Component is the only caller; the client only reads serialized props.

EmptyState is a server-rendered presentational component (no `'use client'`).
EpisodeRow is a server-rendered presentational component (no `'use client'`) — it is rendered as `EpisodeList`'s children mapped from props.
</server_client_split>
