<objective>
Server-side helper that loads `channels.yaml` once via the shipped `@golazo/cli/dist/config/channels.js` loader and exposes a `{ [kidKey: string]: hexAccent }` map for Plan 03's Server Component to pass into list rendering as a prop.

Purpose: UI-02 requires per-kid accent color chips. The accent hex lives in `channels.yaml`. Loading it server-side (a) avoids shipping yaml parser to the client, (b) lets Plan 03 pass the typed map as a serializable prop to its client component, (c) means a missing channels.yaml does NOT crash the list view — it falls back to a single accent for all kids.

Output: One pure async function + one synchronous lookup, both server-only (uses fs via the cli loader). Plan 03's `page.tsx` calls `getChannelAccents()` once at request time; the resulting map is the source of truth for the kid-name chip color in `EpisodeRow`.
</objective>

From @golazo/cli/dist/config/channels.js (loadChannelsFile signature):
```typescript
export function loadChannelsFile(
  opts?: { path?: string; skipTokenCheck?: boolean }
): ChannelsFile;
// ChannelsFile = { [kidKey: string]: ChannelConfig }
// ChannelConfig = { kid, name, club, jersey, accent: string (#RRGGBB), source, youtube: { channelId, oauthTokenPath } }
```

CRITICAL: pass `skipTokenCheck: true` — the UI does NOT need the OAuth token file to exist on disk; it only needs the accent hex. Without this flag, missing token files (an expected state for a fresh dev environment) would throw ChannelsConfigError.

From web/src/theme/tokens.ts (lines 23-30):

```typescript
  // ...
} as const;
```

Fallback contract: `accentFor(map, kidKey)` returns `map[kidKey] ?? COLORS.accentDefault`. The UI never renders an undefined color.
