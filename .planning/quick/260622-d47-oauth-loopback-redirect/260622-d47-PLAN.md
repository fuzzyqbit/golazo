<objective>
Replace the deprecated Google OOB OAuth redirect (`urn:ietf:wg:oauth:2.0:oob`, blocked by Google in 2023) with the loopback-IP flow Google recommends for native/Desktop clients. `runAuth` starts an ephemeral `node:http` server on `127.0.0.1:0`, builds the consent URL with `redirect_uri=http://127.0.0.1:<port>`, and captures the authorization code from the browser redirect instead of prompting the operator to paste it on stdin.

Purpose: The current flow is dead — Google rejects OOB authorization requests, so `golazo auth <kid>` cannot complete. The loopback flow is the supported successor and requires no port pre-registration (Desktop OAuth clients permit ANY loopback port).

Output: Updated `src/publish/oauth.ts` (loopback capture, OOB removed), updated unit + integration tests. No new runtime dependencies — `node:http` only. No docs.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → loopback server | Google redirects the operator's browser to http://127.0.0.1:<port>/?code=... — query string is attacker-influenceable only if a local process forges a request to the bound port during the auth window |
| CLI → Google token endpoint | exchangeCode posts the captured code; tokens returned are secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-d47-01 | Information Disclosure | captureLoopbackCode request handler + defaultReadCode | mitigate | Never log the query string, `code`, or token bytes; close-tab HTML page contains no secret. Unit Test D asserts the code never reaches stdout/stderr/console. |
| T-d47-02 | Tampering | loopback bind address | mitigate | Bind explicitly to `127.0.0.1` (loopback only), never `0.0.0.0`; OS-assigned ephemeral port; server closed immediately after first code/error so the window is minimal. |
| T-d47-03 | Denial of Service | leaked ephemeral server on error path | mitigate | try/finally in runAuth closes the server on any throw; Test C asserts port release after capture. |
| T-d47-04 | Spoofing | forged local request hitting the port before Google's redirect | accept | Single-operator local-Mac tool (per PROJECT.md constraints); no other local users assumed. Window is seconds and port is unpredictable (ephemeral). State-param/PKCE hardening is out of scope for this OOB-replacement task. |
| T-d47-SC | Tampering | npm/pip/cargo installs | accept | No new runtime dependencies introduced — `node:http`/`node:net` built-ins only. No package install tasks in this plan. |
