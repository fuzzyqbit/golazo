# Milestone v1.0 Requirements: golazo

**Archived:** 2026-05-19 (milestone close)
**Defined:** 2026-05-13
**Status:** ✅ 28/28 v1 requirements shipped (1 documented override on PUB-05)
**Core Value:** Drop a folder of clips on disk, get a cinematic per-game highlight episode uploaded to the right YouTube channel — minimal hands-on time per game even at 5+ games/week.

## v1 Requirements (all complete)

### CLI

- [x] **CLI-01**: Operator can run `golazo prepare <folder>`, `golazo render <folder>`, `golazo publish <folder>` as separate idempotent subcommands; each safe to re-run and only acts when needed
- [x] **CLI-02**: Operator can run `golazo all <folder>` to chain prepare → render → publish in one invocation
- [x] **CLI-03**: Operator can run `golazo auth <kid>` once per kid to perform YouTube OAuth and persist a refreshable token at the path declared in `channels.yaml`

### Config

- [x] **CFG-01**: Operator can edit a single `channels.yaml` file holding per-kid branding (name, club, jersey, accent hex, footage source) and YouTube channel binding (channel id, oauth token path)
- [x] **CFG-02**: `channels.yaml` validated at load time; invalid hex, jersey out of range, missing token path fail with clear remediation message

### Prepare

- [x] **PREP-01**: Pipeline parses folder name `YYYY-MM-DD_vs_<slug>_<for>-<against>` into `{date, opponent, scoreFor, scoreAgainst, result}`; malformed names fail loudly with expected format echoed
- [x] **PREP-02**: Pipeline derives kid identity (`leo` / `mateo`) from parent directory under `~/golazo/<kid>/...` and rejects unknown kids with list of valid keys
- [x] **PREP-03**: Pipeline discovers ordered clips by numeric filename prefix; rejects folders with zero matching clips and lists skipped files
- [x] **PREP-04**: Pipeline probes each clip with `ffprobe` and records duration; corrupt clips fail loudly by file name
- [x] **PREP-05**: Pipeline picks music track deterministically from curated YouTube Audio Library pool, seeded by `manifestHash` so re-renders are byte-stable
- [x] **PREP-06**: Pipeline handles music vs episode length — trim+fade when track ≥ episode; deterministic re-roll for longer track when shorter; crossfade two passes as final fallback with stdout warning
- [x] **PREP-07**: Pipeline writes `<folder>/.golazo/manifest.json` whose `manifestHash` is sha256 over sorted `(clipFile, clipSha256)` pairs plus folder name (music and render metadata excluded), enabling stable cache invalidation

### Render

- [x] **REN-01**: Remotion `Episode` composition sequences `TitleCard → (ChapterCard → Clip)× → Outro`; first clip plays at 0.5× rate with original audio muted
- [x] **REN-02**: ChapterCard rhythm rendered every clip when total clips ≤ 5, otherwise grouped every 3 clips
- [x] **REN-03**: Cinematic visual grade applied (vignette + filter) and typography uses Cormorant Garamond Italic (display) + Inter (label), self-hosted under `remotion/assets/fonts/`
- [x] **REN-04**: Music ducked under any clip with audible audio and muted entirely during slo-mo first clip
- [x] **REN-05**: Remotion `Thumbnail` composition produces pure-typographic 1280×720 PNG seeded from same manifest
- [x] **REN-06**: Render driver spawns Remotion CLI programmatically, writes `episode.mp4` and `thumb.png` into `<folder>/.golazo/`, and skips when `manifestHash` matches recorded render unless `--force`

### Publish

- [x] **PUB-01**: `publish` uploads `episode.mp4` via YouTube Data API v3 `videos.insert` with `privacyStatus: "unlisted"`, attaches rendered thumbnail, applies title/description templates from manifest + channel config
- [x] **PUB-02**: Title and description templates substitute `{Kid}`, `{Opponent}`, `{scoreFor}`, `{scoreAgainst}`, `{result}`, `{date}`, `{jersey}`, `{club}`, `{source}` from manifest + channel data
- [x] **PUB-03**: Opponent slug pretty-printed via title-case + hyphen-to-space, with acronym allow-list (`sc`, `fc`, `ac`) preserved upper-case (reuses `prettyOpponent` from Phase 2)
- [x] **PUB-04**: OAuth tokens stored per-kid at path declared in `channels.yaml`; tokens refresh silently on use; refresh failures print clear prompt to rerun `golazo auth <kid>`
- [x] **PUB-05** *(with override)*: Network and 5xx failures retry up to 3 times with exponential backoff (1s, 4s, 16s). **Override accepted at milestone close (2026-05-19):** googleapis SDK uses `uploadType=multipart` rather than YouTube's resumable upload session protocol; mid-upload drops re-upload from byte 0 via the retry policy rather than resuming from last chunk. Functionally adequate for ~50-200 MB episodes on home/club networks. Raw-HTTP resumable rewrite is a v2 candidate. See `.planning/milestones/v1.0-ROADMAP.md` and Phase 3 verification override block.
- [x] **PUB-06**: Quota-exhausted responses (HTTP 403 `quotaExceeded`) fail loudly with "rerun tomorrow" remediation hint; no `publish.json` is written
- [x] **PUB-07**: `publish.json` records `videoId`, `watchUrl`, `uploadedAt`, `channelId`, `privacyStatus`; presence of `videoId` short-circuits subsequent runs unless `--force`

### QA

- [x] **QA-01**: Unit tests (vitest, table-driven) cover filename parser, channels loader, music-picker determinism, and title/description renderers
- [x] **QA-02**: Integration tests cover `prepare` against shipped clip fixture, low-resolution `render` end-to-end, and `publish` with `nock`-stubbed YouTube API; coverage ≥ 80% lines on `src/` (shipped at **86.72% lines**)
- [x] **QA-03**: Remotion compositions pinned by committed `renderStill` snapshots (TitleCard frame + Thumbnail) with 1% pixel-diff threshold via pixelmatch

## Out of Scope (unchanged from v1 definition)

| Feature | Reason |
|---------|--------|
| AI player identification or clip extraction | Operator supplies already-filtered clips from Veo Editor / Trace |
| Cross-game compilations (weekly recaps, season montages) | Episode = one game; recap surfaces are future product |
| Mobile or web UI | CLI sufficient for one-operator local workflow |
| Multi-operator support | Single operator (one parent) on one Mac |
| Voiceover, on-screen statistics, in-game commentary | Cinematic typographic style only — score line is the only on-screen data |
| Folder-watcher daemons or cloud execution | Manual local trigger only |
| Cross-OS rendering (Windows / Linux) | macOS-only; ffmpeg path and font installation assume Homebrew layout |
| Public-by-default publishing | Every upload is unlisted; operator promotes manually in YouTube Studio |
| Licensed music (Epidemic Sound, Artlist, custom tracks) | YouTube Audio Library only — zero copyright risk for small-audience channels |
| Cross-game scoring or season tables | Filename-encoded date / opponent / score is the only metadata |
| Sidecar metadata files (yaml/json) | Filename convention chosen instead |
