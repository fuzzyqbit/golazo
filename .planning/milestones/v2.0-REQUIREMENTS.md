# Milestone v2.0 Requirements: golazo Web UI

**Archived:** 2026-06-02 (milestone close)
**Defined:** 2026-05-31
**Status:** ✅ 22/22 v2.0 requirements shipped (1 deferred override on WEB-QA-03 live run)
**Milestone Goal:** Localhost-only web UI for browsing and playing rendered episodes; read-only surface over v1.0 CLI filesystem outputs.

## v2.0 Requirements (all complete)

### Web Foundation

- [x] **WEB-01**: Next.js 16 web app at `web/` subdir with own `package.json`; root + web share types via npm workspaces (`@golazo/cli` workspace name). Type changes to `src/prepare/manifest.ts` etc. picked up by web via workspace symlink — no copying
- [x] **WEB-02**: `npm run web:dev` from repo root starts web app on `127.0.0.1:4173` via Turbopack; `npm run web:start` runs production build. Both bind exclusively to localhost via inline `HOST=127.0.0.1` + CLI `-H 127.0.0.1` flag
- [x] **WEB-03**: Web app refuses to start when `process.env.HOST` is non-loopback via `web/instrumentation.ts` `validateHostBinding` guard with `WEB-03` stderr token (defense-in-depth alongside CLI flag)

### Discovery + Indexing

- [x] **DISC-01**: `scanGolazoRoot(~/golazo)` returns typed `EpisodeIndex[]` for game folders with valid name + `.golazo/manifest.json`; reuses `parseFilename` + `manifestSchema` via `@golazo/cli/dist/...`
- [x] **DISC-02**: Per-game status (`prepared` / `rendered` / `published`) derived purely from filesystem; recomputed each scan, never stored as truth
- [x] **DISC-03**: sqlite at `web/data/index.db` (gitignored) caches scan results via `better-sqlite3 ^12.0`; p95 query latency 0.073ms at 100 rows (gate <50ms); invalidation by regex-peek manifestHash + tracked-file mtime
- [x] **DISC-04**: chokidar 3.6 watcher with 500ms per-folder debounce reflects filesystem changes in UI within 2 s without full rescan
- [x] **DISC-05**: Scanner ignores folders failing `parseGameFolderName`; broken folders surface in `WarningBag` (in-memory, exposed via `/api/debug/discovery`); never silent skips

### Browse Surface

- [x] **UI-01**: Root route `/` lists all indexed episodes; sortable by date/opponent/result/kid; sort persists in URL search params (`?sort=date.desc&kid=leo`); default `date.desc` omitted from URL
- [x] **UI-02**: Each row shows lazy-loaded thumb.png poster (via `/api/asset/[kid]/[game]/thumb.png`), kid name with accent-color chip from `channels.yaml` (loaded server-side with `skipTokenCheck:true`), opponent pretty-printed via Phase 2 `prettyOpponent`, date, score, status badge
- [x] **UI-03**: Per-kid chip filter (`all`/`leo`/`mateo`) reduces list; combined sort + filter state in URL search params for deep-linkability
- [x] **UI-04**: Empty state renders clear message with the scanned root path; not a blank screen
- [x] **UI-05**: Typography matches Remotion compositions — Cormorant Garamond Italic display + Inter labels via `next/font/local` resolving same TTF bytes as Remotion (no duplication)
- [x] **UI-06**: Layout desktop-optimized (operator MacBook); mobile breakpoints not pinned by tests

### Episode Detail + Playback

- [x] **PLAY-01**: Route `/episodes/<manifestHash>` is canonical permalink; unknown hash → 404 via `notFound()` + custom `not-found.tsx`
- [x] **PLAY-02**: Detail view shows rendered title via `renderTitle`, rendered description via `renderDescription`, full manifest details (clip list with durations, music pick info, render block), publish.json contents when present with YouTube Studio link
- [x] **PLAY-03**: When `episode.mp4` exists, HTML5 `<video controls preload="metadata" poster src>` streams via Next.js route handler at `/api/asset/<kid>/<game>/episode.mp4`; thumb.png is poster
- [x] **PLAY-04**: Asset route handler enforces path safety via `assertSafeAssetPath` — request paths resolved against `~/golazo/`; escapes via `..`, symlinks, or absolute paths return 403
- [x] **PLAY-05**: Asset route supports HTTP Range — `Range: bytes=0-99` returns 206 with first 100 bytes via RFC 7233 single-range parser; malformed → 200 full; unsatisfiable → 416

### Quality + Testing

- [x] **WEB-QA-01**: Vitest unit tests cover scanner, sqlite cache, cacheInvalidation, assetPath, rangeParser (5 test files, table-driven, following Phase 1+2 patterns)
- [x] **WEB-QA-02**: Vitest integration tests verify list view + detail view + episode-asset + detail-player against `web/tests/fixtures/golazo/` with 4 game folders spanning all 3 statuses + broken-folder edge case
- [x] **WEB-QA-03** *(with deferred override on live run)*: Playwright E2E suite covers golden path (open `/` → filter `leo` → click first episode → `<video>` fires `play` + `currentTime > 0`) + path-traversal regression (`GET /api/asset/../../etc/passwd` → [403, 404]); Chromium-only; baseline coverage **95% lines (418/440)** vs 80% gate (15pp headroom). **Override accepted at milestone close (2026-06-02):** live `npm run web:e2e` browser run requires `npx playwright install chromium` binary fetch + operator-side dev server lifecycle; equivalent surface exercised by integration tests on ports 4175/4176/4177/4178/4179. Tracked as `WEB-E2E-LIVE-RUN` v2.1 pre-public-flip smoke.

## Out of Scope (v2.0)

| Feature | Reason |
|---------|--------|
| Render/publish from browser | Read-only surface in v2.0 — CLI keeps action authority |
| Multi-user auth or sessions | Single operator on one Mac; localhost bind is access control |
| LAN-accessible mode | Carry-forward for v2.1+ if phone access becomes a real need |
| Cross-OS deployment | Operator's Mac is the only target |
| Editing manifest / channels via UI | Filesystem + yaml + CLI remain source of truth |
| Public YouTube discovery / playlist syncing | Out of scope (v1 unlisted-only constraint extends to v2 browse) |
| Mobile-first responsive layouts | Desktop-first; mobile best-effort, not pinned by tests |
