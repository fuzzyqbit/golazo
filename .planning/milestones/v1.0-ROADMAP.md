## Overview

golazo v1.0 ships a local-Mac CLI that turns folders of soccer highlight clips into branded, per-game YouTube episodes for two kids' channels. Built as four horizontal layers: prepare → render → publish → convenience/QA. Each phase delivered a complete, verifiable layer the operator can exercise from the command line before the next layer was built on top.

## Milestone Summary

**Decimal Phases:** none — all four integer phases shipped in order, no urgent insertions.

**Key Decisions:**
- npm chosen over pnpm (pnpm not installed on dev Mac); `packageManager: "npm@10.9.0"` pinned (01-01)
- ESLint flat config (`eslint.config.js`) — ESLint 10 dropped legacy `.eslintrc.cjs` (01-01)
- Stub contract for unimplemented subcommands frozen as `cmd.error('<name>: not yet implemented', { exitCode: 2, code: '<name>.unimplemented' })` — each phase removed the stub for its subcommand and asserted absence via grep gate (01-01 → 02-04 → 03-05 → 04-01)
- `*.test-cases.ts` sibling pattern for table-driven tests (vitest test files cannot be imported under `tsx -e`) (01-02)
- `noUncheckedIndexedAccess: true` throughout — config loaders use `if (!entry) continue` guards even on zod-validated records (01-02)
- manifestHash canonical input is `folderName + '\n' + sorted "file:sha256" lines` — invariant across music/render/publish block additions (01-04, preserved through 02-02, 02-04)
- Fixture HOME stub pattern (`vi.stubEnv('HOME', process.cwd())` and `HOME=$PWD npx tsx ...`) for tilde-pathed `oauth_token` (01-04)
- CLI shell-out integration test pattern (`promisify(execFile)('npx', ['tsx', 'src/cli/index.ts', ...])` with HOME forwarded) — reused unchanged across Phases 2/3/4 (01-05)
- `@remotion/fonts` `loadFont()` (single face per call) — not `loadFonts()`; three calls in `Promise.all()` for Cormorant Garamond Italic + Inter Regular + Inter Bold (02-01)
- `tsconfig.check.json` separate config for typecheck — base `tsconfig.json` rootDir stays `./src` so `dist/cli/index.js` bin path is preserved; check config sets `rootDir: '.'` + `noEmit: true` + includes `src/**/*` + `remotion/**/*` (02-01)
- Music-pick seed derivation pinned: `sha256(manifestHash + ':roll:' + r).slice(0,16)` → `Number(BigInt('0x' + hex) % BigInt(pool.length))` — byte-stable across machines via Node `crypto` + BigInt modulo (02-02)
- `musicVolumeAtFrame` extracted as pure unit-tested function, imported by `Episode.tsx` (grep-gated against redefinition) (02-03 revision)
- googleapis SDK auto-selects `uploadType=multipart` (not resumable) for stream + requestBody — pinned by test case 6; retry-from-zero policy in `withRetry` handles transient failures (03-03)
- `PRIVACY_STATUS = 'unlisted' as const` exported from uploader; `publishRecordSchema` uses `z.literal('unlisted')` + compile-time `satisfies typeof PRIVACY_STATUS` bridge — defense in depth at source, type, and schema layers (03-03 revision + 03-05 revision)
- `GOLAZO_OAUTH_MOCK=1` test-only env-var shim in `exchangeCode` — Phase 4 carry-forward to remove via injectable factory pattern (03-01)
- Self-hosted fonts under `remotion/assets/fonts/` eliminate system-font drift for snapshot determinism (02-01 → 04-04)

**Issues Resolved:**
- Phase 2 verification "human_needed" items (cinematic grade visible, chapter-rhythm visual difference, music ducking audible) — structurally closed by Phase 4 QA-03 renderStill PNG baselines + pixelmatch 1% gate + pure unit tests on `musicVolumeAtFrame` and `computeChapterRhythm`
- googleapis upload-protocol spec drift (resumable vs multipart) — documented + accepted via Phase 3 override block; v2 candidate

**Issues Deferred / Tech Debt (v2 carry-forward):**
- **PUB-05 resumable upload**: googleapis SDK uses `uploadType=multipart`, not the YouTube resumable session protocol. Multipart + 3-retry-from-zero is functionally adequate for typical episode sizes (~50-200 MB) on home/club networks; revisit if real-world fail rates warrant a raw-HTTP rewrite.
- **GOLAZO_OAUTH_MOCK shim removal**: replace test-env-var branch in `src/publish/oauth.ts` `exchangeCode` with an injectable implementation (dependency injection via constructor or factory).
- **Chapter-rhythm visual regression coverage**: pure-logic + 3-clip baseline are pinned; a 6+-clip fixture rendered to PNG would close the visual variant. Low-risk; logic gate already in place.
- **5 outstanding git stashes** accumulated across phase boundaries (pre-existing src/ refactor mods) — review with `git stash list` and merge or drop after v1.0 close.

**Technical Debt Incurred:**
