## Substitution Recipe

To replace a test-tone fixture with a real YouTube Audio Library track:

1. Download the track from YouTube Studio â†’ Content â†’ Audio Library
2. Drop the `.mp3` into `remotion/assets/music/`, renamed to `<slug>.mp3`
   (lowercase ASCII + hyphens only, matching `^[a-z0-9-]+\.mp3$`)
3. Append or update the matching entry in `index.json`:
   ```json
   { "file": "<slug>.mp3", "title": "<track title>", "durationSec": <actual seconds>, "mood": "<atmos|driving|uplift|tense>" }
   ```
4. Commit both the `.mp3` and the updated `index.json`.

The picker (`src/render/musicPicker.ts`) uses **only** the `index.json` entries â€”
files present in the directory but not declared in the index are silently ignored.

## Determinism Note

The fixture sine tones are generated with a deterministic ffmpeg recipe. Regenerating
them with the recipe below produces byte-identical files across machines (no
x264-style threading nondeterminism for audio-only encodes):

```bash
mkdir -p remotion/assets/music
set -e
declare -a DUR=(200 60 240 30 180 8)
declare -a NAME=(atmos-1 atmos-2 atmos-3 atmos-4 atmos-5 atmos-6)
for i in 0 1 2 3 4 5; do
  ffmpeg -y -f lavfi -i "sine=frequency=$((220+50*i)):duration=${DUR[$i]}" \
    -ac 1 -ar 22050 -b:a 32k \
    -metadata title="${NAME[$i]} (test tone)" \
    -metadata artist="golazo-fixture" \
    "remotion/assets/music/${NAME[$i]}.mp3"
done
```

After regenerating, run `ffprobe` on each file and update `index.json` durations to match
the reported values (rounded to 3 decimals) â€” ffmpeg's MP3 encoder may pad frames
slightly, so the reported duration is always slightly above the requested duration.
